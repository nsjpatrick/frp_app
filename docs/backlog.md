# Post-V1 Backlog

Deferred items from the final code review of Plan 1 + Plan 2. Each line notes the recommended plan to land the fix.

## Deferred from code review 2026-04-20

- **Tenant-isolation integration tests** — seed two tenants, authenticate as each user, verify neither can read/write the other's customer / project / quote / revision through any server action or route. The `createProject` cross-tenant bug (fixed in this commit) is exactly what such tests would have caught. **Land in Plan 6 (Admin & RBAC).**
- **Race in `recomputeStructuralAnalysis` outputs merge** — `lib/actions/revisions.ts` does read → merge → write on `revision.outputs`. Two concurrent step saves can clobber each other. Fix with `jsonb_set` in a single SQL statement or wrap in a serializable transaction. **Land in Plan 3.**
- **Unvalidated JSON hidden-field parse** — `saveServiceStep` calls `JSON.parse` on `ansiStandards`, `requiredDocuments`, and `siteJson` from form data. Malformed input throws raw SyntaxError. Wrap in try/catch and surface a structured validation error. **Plan 3 (same file gets touched for pricing).**
- **Anchor catalog-exhaustion fallback assumes sorted order** — `lib/rules/anchor-sizing.ts` falls back to `catalog[catalog.length - 1]` when every anchor size exceeds `MAX_QTY_PER_SIZE`. Not reachable with the current 5-entry seed, but fragile. Sort by `capacityLbfEach` descending in the fallback, or replace the fallback with `reduce(max)`. **Plan 3 (catalog-versioning work touches this file).**
- **Revision immutability after `sent`** — spec §4 says "A Revision's PricingSnapshot is immutable once the Quote is `sent`. Edits produce a new Revision." Not enforced in `saveServiceStep` / `saveGeometryStep` / `saveResinStep`. Add a check that throws if `rev.quote.status !== 'DRAFT'`, and add a "new revision" action that clones the current rev with an incremented label. **Plan 3 (pricing engine needs this invariant to be real).**
- **AuditLog DB role not ACL-restricted** — trigger blocks UPDATE/DELETE for everyone, but the app still connects as schema owner. Spec §4 says "the DB role used by the app has INSERT only on that table." Add a migration that creates a role `app_writer` with `GRANT INSERT` on `AuditLog` and other tables' SELECT/INSERT/UPDATE/DELETE as appropriate. **Plan 6.**
- **`session?.user as any` repeated in 9+ files** — collapse into one `requireSessionUser()` helper that returns a typed `SessionUser`. Removes casts and undefined-safety footguns across `lib/actions/*.ts` and all `app/(app)/**/page.tsx`. **Plan 3 (touching these files for pricing anyway).**
- **`AnchorSizingResult.pitch` field is vestigial** — always hardcoded to `'acceptable'`; the `'inches'` variant never set. Either compute a real bolt-spacing check from the tank circumference, or drop the field. **Plan 5 or delete now.**
- **Engineering JSON `generated_at` non-deterministic at runtime** — the route handler omits `generatedAt`, so every download uses `new Date().toISOString()`. Harmless while `checksum_sha256: null`, but once checksums land in Plan 4 the digest will differ every fetch. Use `rev.updatedAt.toISOString()` when the caller doesn't pass a pin. **Plan 4 (PDF output + checksums).**
- **`/api/test/login` production guard test** — the guard checks `NODE_ENV === 'production'` but there's no test proving it. Add a small test. **Plan 3 or any passing touch.**
- **Concurrent-recompute test** — race two step-save server actions; assert both updates land. Same file as the outputs-merge fix above. **Plan 3.**
- **Anchor sizing catalog-exhaustion test** — `sizeAnchors({ upliftLbf: 10_000_000, catalog: SEED_ANCHORS })` has no coverage of the fallback path. **Plan 3.**
- **Engineering JSON determinism assertion** — existing test only proves identical when `generatedAt` is pinned. Add a test that asserts identical bytes for two calls when `generatedAt` is omitted (after the fix above). **Plan 4.**
- **Structural-recompute audit entry** — `recomputeStructuralAnalysis` persists a new `outputs.structuralAnalysis` but does not write an audit entry describing the recomputation. For ISO 9001 design-change traceability, emit a separate `action: 'recompute:structural'` audit row with the key outputs. **Plan 6.**
- **`auth.ts` session callback double fetch** — every request hits `db.user.findUnique` to pull role + tenantId. Fine for the pilot; at traffic, store these on the Session row via adapter extension. **Plan 6.**
- **Wizard layout re-invokes `auth()`** — the `(app)` layout already guards, but every child page calls `auth()` again. Use React's `cache()` or pass the session via context. **Plan 6.**
- **Load-combination lever-arm assumption** — `EFFECTIVE_LEVER_ARM_FRACTION_OF_D = 0.4` is a conservative approximation. Emit it as an output field (`loadCombination.leverArmFractionOfD`) so PE reviewers see which assumption drove the uplift calc. **Plan 4 (output shape polish).**
- **Dead constants** — `D3299_AXIAL_TO_HOOP_THICKNESS_RATIO` and `FT_PER_IN` are exported but never referenced. Trim. **Plan 3 or any opportunistic edit.**
